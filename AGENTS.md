# JM Infrastructure Project Instructions

## Project identity

This project is the official website for JM Infrastructure.

JM Infrastructure builds digital systems and online infrastructure for businesses. The site must not present the business as simply “a website company”.

Websites are one part of the offer, alongside search presence, credibility, enquiry flow, follow-up systems, review generation, and operational clarity.

Core positioning:
Many businesses are better in reality than they appear online. JM Infrastructure fixes the gap between real-world quality and online representation.

## Audience language

Use broad business language.

Preferred terms:
- businesses
- local businesses
- service businesses
- owner-led businesses
- reputation-led businesses
- businesses built on trust

Do not mention specific industries such as salons, trades, barbers, electricians, decorators, roofers, or landscapers unless explicitly requested.

## Copy rules

Keep copy minimal.

Most text blocks should be one line or two short sentences maximum.

Avoid long paragraphs.

Avoid generic agency language, hype, and filler.

Do not use phrases like:
- digital transformation
- unlock your potential
- scale your brand
- bespoke solutions
- take your business to the next level

Copy should feel sharp, commercial, grounded, human, and restrained.

## Design direction

The site should feel serious, premium, structured, professional, and systems-focused.

It should not feel like:
- a generic web design agency
- a SaaS landing page
- an AI hype site
- a template portfolio
- an overanimated creative experiment

Use restrained cinematic movement only where it improves comprehension.

No scroll hijacking.

No decorative motion that delays reading.

No heavy animation libraries unless explicitly approved.

## Technical stack

Use:
- Next.js
- TypeScript
- Tailwind CSS
- App Router
- semantic HTML
- mobile-first layout

Keep dependencies minimal.

Ask before adding new production dependencies.

## SEO and launch standards

Preserve SEO, accessibility, and performance.

Requirements:
- clear page title
- meta description
- canonical URL
- Open Graph metadata
- JSON-LD structured data where appropriate
- robots.txt
- sitemap.xml
- one clear h1 per page
- logical h2/h3 structure
- semantic HTML
- readable text content, not text hidden inside images
- descriptive image alt text
- optimised images
- accessible contrast
- fast loading
- no broken links
- no accidental noindex settings

The homepage must clearly communicate:
- JM Infrastructure
- digital systems / online infrastructure
- search presence
- first impression
- credibility
- enquiry flow
- follow-up / reviews
- Glasgow / Central Scotland where natural

Do not keyword-stuff.

## Workflow rules

Do not jump straight into large builds.

For major work:
1. inspect existing files
2. explain the proposed plan briefly
3. wait for approval if the change is broad
4. make focused changes
5. summarise what changed
6. run practical checks where possible

For this project, separate these stages:
1. extract taste profile from reference site
2. create design/build plan
3. build first version
4. audit
5. refine

Do not clone the Decor reference site. Extract transferable design principles only.

## Skill usage

Use relevant installed skills when appropriate:

- jm-taste-profile-extractor for inspecting previous site ZIPs, screenshots, or design references.
- jm-positioning-copy-guard for copy, positioning, section text, CTAs, and offer language.
- jm-seo-launch-audit for SEO, accessibility, performance, metadata, and pre-launch checks.

If a task matches a skill, use the skill before making changes.
